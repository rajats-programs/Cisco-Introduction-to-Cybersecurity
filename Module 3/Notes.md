# Module 3: Protecting Your Data and Privacy

## 3.1 Protecting Your Devices and Network

## Protecting Your Computing Devices

Computing devices store valuable personal and sensitive information. Protecting these devices prevents unauthorized access, malware infections, and data theft.

Examples of computing devices:
- Computers
- Smartphones
- Tablets
- Smart devices

### Security Practices:

### 1. Keep Software Updated
- Updates fix security vulnerabilities.
- Outdated software can be exploited by attackers.

### 2. Use Security Software
Examples:
- Antivirus
- Firewall
- Anti-malware tools

Purpose:
- Detect threats
- Block malicious activities
- Protect system resources

### 3. Secure Physical Access
- Use screen locks
- Avoid leaving devices unattended
- Protect devices from theft

### 4. Download Carefully
- Install applications only from trusted sources.
- Avoid suspicious files and links.

---

# Wireless Network Security at Home

A wireless network connects devices using Wi-Fi technology.

Security risks:
- Unauthorized users accessing your network
- Data interception
- Network attacks

### Ways to Secure Home Wi-Fi:

- Change default router username and password
- Use WPA2 or WPA3 encryption
- Create a strong Wi-Fi password
- Update router firmware
- Disable unnecessary features

---

# Public Wi-Fi Risks

Public Wi-Fi networks are available in places like:

- Airports
- Cafes
- Hotels
- Libraries

Security risks:

- Attackers can monitor network traffic
- Fake Wi-Fi networks can steal information
- Sensitive data can be exposed

### Safety Practices:

- Avoid online banking on public Wi-Fi
- Use HTTPS websites
- Use VPN when necessary
- Disable automatic Wi-Fi connections

---

# Password Security

Passwords are the first line of defense for protecting accounts.

A weak password can allow attackers to:

- Access accounts
- Steal information
- Perform identity theft

---

# Creating a Strong Password

A strong password should have:

- Long length
- Combination of characters
- Numbers
- Symbols
- No personal information

Weak example:

rajat123


Strong example:

R@7tK!92mQx


Avoid:

- Names
- Birth dates
- Common words
- Reusing passwords

---

# Using a Passphrase

A passphrase is a long password created using multiple words.

Example:


CoffeeRainLaptopMountain


Advantages:

- Easier to remember
- Harder to crack
- More secure due to length

---

# Password Guidelines

Best practices:

- Use unique passwords for every account
- Enable Multi-Factor Authentication (MFA)
- Change compromised passwords immediately
- Do not share passwords
- Use a password manager
- Avoid saving passwords on public computers

---

# 3.2 Data Maintenance

## What is Encryption?

Encryption is the process of converting readable data (plaintext) into an unreadable format (ciphertext) to prevent unauthorized access.

Process:


Plaintext → Encryption → Ciphertext


To access the original data:


Ciphertext → Decryption → Plaintext


---

# Importance of Encryption

Encryption provides:

- Confidentiality
- Data protection
- Secure communication

Used in:

- Online banking
- Messaging apps
- Secure websites

---

# Types of Encryption

## Symmetric Encryption

Uses the same key for encryption and decryption.

Advantages:
- Faster

Disadvantage:
- Key sharing can be difficult

---

## Asymmetric Encryption

Uses two keys:

- Public key
- Private key

Advantages:
- More secure for communication

Example:
- HTTPS communication

---

# How to Encrypt Your Data

Methods:

- Enable device encryption
- Use encrypted storage
- Use secure communication apps
- Protect files with encryption tools

Examples:

- Full disk encryption
- Encrypted cloud storage

---

# Backup Your Data

A backup is a copy of important data stored separately.

Purpose:

- Recover lost files
- Protect against ransomware
- Prevent permanent data loss

Backup locations:

- External drives
- Cloud storage
- Network storage

---

# Backup Best Practices

Follow the 3-2-1 Backup Rule:

- 3 copies of data
- 2 different storage types
- 1 copy stored offline

---

# Are Data Really Gone?

Deleting a file does not always permanently remove it.

Normal deletion:
- Removes file reference
- Data may still exist and can be recovered

Attackers can use recovery tools to restore deleted files.

---

# Permanently Deleting Data

Methods:

- Secure erase tools
- Data wiping software
- Encryption before deletion

These methods overwrite data so it cannot be recovered.

# 3.4 Safeguarding Your Online Privacy

## Two-Factor Authentication (2FA)

Two-factor authentication adds an extra security layer by requiring two forms of verification.

Factors:

1. Something you know
   - Password

2. Something you have
   - Phone or security key

3. Something you are
   - Fingerprint or face recognition

Benefits:

- Protects accounts even if passwords are stolen

---

# Open Authorization (OAuth)

OAuth is a secure authorization method that allows applications to access limited information without sharing passwords.

Example:

Logging into an app using a Google account.

Benefits:

- Reduces password exposure
- Gives controlled access

---

# Social Sharing

Sharing too much information online can create security risks.

Attackers may use:

- Birth dates
- Locations
- Personal interests

for:

- Identity theft
- Social engineering attacks

Safety:

- Think before posting
- Limit personal information
- Review privacy settings

---

# Don't Get Spoofed

Spoofing is when attackers pretend to be a trusted person, website, or organization.

Types:

## Email Spoofing
Fake emails pretending to come from trusted sources.

## Website Spoofing
Fake websites designed to steal information.

## Caller ID Spoofing
Fake phone numbers used by attackers.

Protection:

- Verify sources
- Avoid suspicious links
- Check website addresses

---

# Email and Web Browser Privacy

Email security:

- Avoid unknown attachments
- Check sender addresses
- Use spam filters

Browser security:

- Keep browsers updated
- Use secure websites (HTTPS)
- Avoid suspicious extensions
- Clear unnecessary cookies
